✔ Network Security Created SIEM tools on Microsoft Azure with Microsoft Sentinel for real-time visibility and incident response.

  • 1. Create a Windows Virtual Machine on Azure.
  • 2. Create a resource group, which serves as the container for all the objects we're creating.
  • 3. Select the Windows 10 Pro ISO image for the virtual machine.
  • 4. Set the administrative username and password for access control.
  • 5. Select the "Allow Selected Ports" option, enabling RDP (3389) for network traffic.
  • 6. Choose disk size and networking options based on your requirements.
  • 7. Review the configuration and deploy the virtual machine if satisfied.
  • 8. Ensure your virtual machine settings align with security best practices.
  • 9. Deploy Microsoft Sentinel and link the resource group for resource tracking.
  • 10. Name and configure the Log Analytics workspace, waiting for logs to be generated.
  • 11. Add the virtual machine to the event logs, forwarding logs to Log Analytics and linking to Microsoft Sentinel.
  • 12. Set up a data connector within the Microsoft Sentinel dashboard.
  • 13. Install Windows Security Events via the Content Hub in Microsoft Sentinel.
  • 14. Open the data connector and configure the necessary rules for data collection.
  • 15. Verify that logs are being collected successfully from the connected systems.
  • 16. Navigate to Microsoft Sentinel's log terminal, type "SecurityEvent", and analyze the data.
  • 17. Create a rule in Microsoft Sentinel to log all successful logins: | where Activity contains "success".
  • 18. Create additional rules and assign severity levels based on the criticality of events.
  • 19. Review all created rules within the Microsoft Sentinel Analytics page.
  • 20. Log back into the system and verify alerts are triggered as configured, with severity levels applied.
